01. What is C#?

Think of C# (pronounced "C-sharp") as Microsoft's answer to Java. It's a modern, object-oriented programming language that lets you build just about anything — from websites and desktop apps to mobile apps and video games. Microsoft created it, and some of the biggest names in tech (Microsoft, Stack Overflow, Unity) rely on it every single day.

The latest version is C# 12.0, and it keeps getting better with each release. If you already know a bit of C, C++, or Java, you'll feel right at home here.

02. Writing Your First Program

Every programmer's journey starts with "Hello, World!" — it's a tradition. Here's what that looks like in C#. Don't worry if the syntax looks unfamiliar; we'll break everything down in later sections.

using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}
Hello, World!
💡
What's happening here? The using System; line imports built-in tools. namespace organizes your code. class is a container for methods. Main() is the starting point — the very first thing that runs when you hit "Run".

03. A Quick History

C# was born out of necessity. In the late 1990s, Microsoft needed a language that combined the power of C++ with the simplicity of Visual Basic. Anders Hejlsberg (the brilliant mind who also created Turbo Pascal and TypeScript) led the team that designed C#.

  • Developed under Microsoft's .NET initiative
  • Standardized by ECMA and ISO — so it's not just a Microsoft thing
  • First version released in 2002 alongside Visual Studio .NET
  • Closely related to C/C++ and Java in syntax and philosophy
⚠️
Keep in mind: C# 12.0 (released November 2023) is the latest version. Each new version introduces powerful features that make coding easier and more expressive.

04. .NET Framework & Its Components

Before your C# code can run on a computer, it needs a "platform" to live on. That platform is .NET — a software framework by Microsoft that acts like a middleman between your code and the operating system. Think of it as a translator that takes your C# instructions and turns them into something the computer actually understands.

.NET supports 60+ programming languages, but C# and VB.NET are the stars of the show. The first version (1.0) shipped in 2002.

Common Language Runtime (CLR)

The CLR is the engine of .NET. When you run a C# program, the CLR is what actually executes it. It doesn't care which .NET language you wrote in — it treats them all the same. It also handles memory management, security, and error handling behind the scenes.

  • Managed Code — Code that runs inside the CLR. It gets perks like garbage collection and type safety.
  • Unmanaged Code — Code that runs directly on the OS, bypassing the CLR. Faster but riskier.

Framework Class Library (FCL)

Imagine having a massive toolbox filled with pre-built tools for every common task — reading files, connecting to databases, making web requests. That's the FCL. It's a huge collection of reusable classes, methods, and interfaces you can plug into your code instead of building everything from scratch.

Garbage Collector (GC)

In languages like C++, you have to manually free memory when you're done with it (and forgetting to do so causes memory leaks). C# solves this problem with a Garbage Collector that automatically detects objects you're no longer using and reclaims that memory. It runs on the Managed Heap with an internal optimization engine.

Just-In-Time (JIT) Compiler

Here's how your code actually becomes executable: First, the C# compiler converts your source code into CIL (Common Intermediate Language) — a set of platform-independent instructions. Then, at runtime, the JIT compiler converts that CIL into native machine code optimized for your specific computer. It's like having a translator who speaks your computer's exact dialect.

05. Visual Studio

Visual Studio is Microsoft's all-in-one IDE (Integrated Development Environment). It's where you write, debug, and test your C# code. Think of it as a super-powered text editor that understands your code, highlights errors in real-time, and even suggests fixes.

  • Supports 36+ languages (C#, C++, Python, JavaScript, etc.)
  • Available for Windows and macOS
  • Community Edition is completely free for learning and small teams
✅
Quick setup: Download Visual Studio Community → Run installer → Select ".NET Desktop Development" workload → Create a new Console App project → Start coding!

With so many languages out there, why pick C#? Here are the honest reasons:

  • Gentle learning curve — If you've seen any C-like language before, C# will feel familiar. Even beginners pick it up quickly.
  • Versatile — One language for web apps (ASP.NET), desktop apps (WPF/WinForms), mobile apps (Xamarin/MAUI), and games (Unity).
  • Massive community — Millions of developers, endless tutorials, and StackOverflow answers for every question you'll ever have.
  • Game development powerhouse — Unity (the world's most popular game engine) uses C# as its primary language. Many indie and AAA games are built with it.
  • Strong job market — Enterprise companies love C# for its reliability and Microsoft ecosystem integration.

07. C# Fundamentals

Identifiers

An identifier is simply a name you give to things in your code — your classes, methods, variables, etc. Think of them as labels on boxes: the label tells you what's inside.

Rules to remember:

  • Can contain letters (A-Z, a-z), digits (0-9), and underscore (_)
  • Must NOT start with a digit (so 123name is invalid)
  • Cannot be a reserved keyword (unless you prefix it with @)
  • Case-sensitive: myVar and MyVar are different
  • Max 512 characters (you'll never hit this limit in practice!)
public class Student {        // "Student" is an identifier (class name)
    static void Main()         // "Main" is an identifier (method name)
    {
        int age = 21;            // "age" is an identifier (variable name)
    }
}

Keywords

Keywords are reserved words that C# has claimed for itself. They have special meanings (like if, class, return) and you can't use them as your own variable or class names. C# has 78 keywords in total.

Common ones: using, namespace, class, public, private, static, void, int, string, bool, if, else, for, while, return, new, this, null, true, false.

Variables

A variable is like a labeled container that holds data. You put something in it, give it a name, and refer to it later whenever you need that data.

// Syntax: type name = value;
char grade = 'A';        // holds a single character
int score = 95;          // holds a whole number
double price = 19.99;    // holds a decimal number
string name = "Alice";   // holds text
bool isPassed = true;    // holds true or false

Naming rules: Can contain letters, digits, and _. Cannot start with a digit. Cannot be a keyword.

Two ways to initialize: Compile-time (value set in code) or Run-time (value comes from user input or calculation during execution).

Literals

Literals are fixed values written directly in your code. When you write int x = 100;, the 100 is a literal — it's a concrete value, not a variable.

Enumerations (enum)

Enums let you create a named set of constants. Instead of remembering that "0 means Sunday, 1 means Monday," you can write Day.Sunday and your code becomes self-documenting.

enum Month { Jan, Feb, Mar, Apr, May }
// Jan=0, Feb=1, Mar=2, Apr=3, May=4 (auto-assigned)

Console.WriteLine((int)Month.Mar);  // Output: 2
📚
By default, the first member starts at 0 and each subsequent one increments by 1. You can override this: enum Month { Jan = 1, Feb, Mar } would make Jan=1, Feb=2, Mar=3.

08. Decision Making Statements

Just like in real life, your program needs to make choices. "If the user is logged in, show the dashboard. Otherwise, show the login page." This is what decision-making statements do — they let your code take different paths based on conditions.

The main types: if, if-else, if-else-if ladder, and nested if.

int i = 20;

if (i == 10)
    Console.WriteLine("i is 10");
else if (i == 15)
    Console.WriteLine("i is 15");
else if (i == 20)
    Console.WriteLine("i is 20");     // This runs!
else
    Console.WriteLine("not found");
i is 20

09. Switch Statement

When you have many possible values to check, an if-else-if chain gets ugly fast. The switch statement is a cleaner alternative — it takes one expression and jumps directly to the matching case.

📚
Rules: No duplicate cases. Case values must be constants (not variables). Every case needs a break (or return / goto). The default case is optional but recommended.
int day = 3;
switch (day) {
    case 1: Console.WriteLine("Monday");  break;
    case 2: Console.WriteLine("Tuesday"); break;
    case 3: Console.WriteLine("Wednesday"); break;
    default: Console.WriteLine("Other"); break;
}
Wednesday

10. Loops

Loops are how you tell the computer to repeat something. Instead of writing the same line 100 times, you write it once inside a loop. There are two families:

  • Entry-controlled (while, for) — Condition checked first. If false from the start, body never runs.
  • Exit-controlled (do-while) — Body runs at least once, then condition is checked.

while Loop

"Keep going while this is true."

int x = 1;
while (x <= 4) {
    Console.WriteLine("Count: " + x);
    x++;
}

do-while Loop

"Do this first, then check if you should keep going." Always executes at least once.

int x = 100;
do {
    Console.WriteLine("Runs once!");
    x++;
} while (x < 5);  // condition is false, but body already ran

for Loop

Best when you know exactly how many times you want to repeat.

for (int i = 0; i < 5; i++)
    Console.WriteLine("Iteration: " + i);

foreach Loop

The friendliest loop — walks through every item in a collection without needing an index.

int[] nums = { 10, 20, 30, 40 };
foreach (int n in nums)
    Console.WriteLine(n);

11. Jump Statements

Sometimes you need to break out of the normal flow — exit a loop early, skip an iteration, or jump somewhere else entirely.

break

Immediately exits the loop or switch it's inside. Like hitting the emergency stop button.

for (int i = 1; i <= 10; i++) {
    if (i == 5) break;        // stops at 5
    Console.Write(i + " ");     // prints: 1 2 3 4
}

continue

Skips the rest of the current iteration and jumps to the next one. Like saying "skip this, move on."

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;     // skips 3
    Console.Write(i + " ");     // prints: 1 2 4 5
}

goto

Jumps to a labeled statement. Useful in switch blocks but generally discouraged elsewhere (it makes code hard to follow).

return

Exits the current method and sends a value back to whoever called it. If the method is void, return; simply exits without sending anything.

12. Arrays

An array is a collection of items of the same type, stored in a row in memory. Think of it as a row of mailboxes — each box has a number (index) starting from 0.

string[] days = { "Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat" };

Console.WriteLine(days[0]);      // Sun (index 0)
Console.WriteLine(days.Length);  // 7 (total elements)

foreach (string d in days)
    Console.Write(d + " ");
Sun Mon Tue Wed Thu Fri Sat
⚠️
Array sizes are fixed once created. If you need a resizable collection, use a List<T> instead.

13. Strings

A string is simply text — a sequence of characters. In C#, strings are objects of the System.String class and they are immutable (once created, they can't be changed; any "modification" actually creates a new string).

string s1 = "Hello";            // using 'string' keyword
String s2 = "World";            // using 'String' class
string full = s1 + " " + s2;    // concatenation: "Hello World"
💡
string (lowercase) and String (uppercase) are identical — both are aliases for System.String. Use whichever you prefer, but string is the convention.

15. Object-Oriented Programming (OOP)

OOP is a way of organizing code around objects (things) rather than actions. Instead of writing one giant procedure, you create classes that model real-world entities, each with their own data and behavior. The four pillars are: Encapsulation, Inheritance, Polymorphism, and Abstraction.

Class & Object

A class is a blueprint (like an architect's plan for a house). An object is a specific instance built from that blueprint (an actual house you can live in). You define the structure once in the class, then create as many objects as you need.

public class Dog {
    public string Name;
    public int Age;

    public Dog(string name, int age) {
        Name = name; Age = age;
    }
}

Dog buddy = new Dog("Buddy", 3);
Console.WriteLine(buddy.Name);  // Buddy

Constructors

A constructor is a special method that runs automatically when you create a new object. Its job is to set up the object's initial state. It has the same name as the class and no return type.

Destructors

The opposite of constructors — they run when an object is about to be garbage collected. Defined with ~ClassName(). You rarely need to write one because the Garbage Collector handles cleanup automatically.

Inheritance

Inheritance lets you create a new class based on an existing one. The child class gets all the parent's features for free, and can add its own on top. It's like saying "a Dog is an Animal" — it inherits all animal behaviors and adds dog-specific ones.

class Animal {
    public void Eat() { Console.WriteLine("Eating..."); }
}

class Dog : Animal {          // Dog inherits from Animal
    public void Bark() { Console.WriteLine("Woof!"); }
}

Dog d = new Dog();
d.Eat();   // inherited from Animal
d.Bark();  // defined in Dog

Encapsulation

Encapsulation means hiding the internal details and only exposing what's necessary. You make variables private and provide controlled access through properties (get/set). It's like a vending machine — you interact through buttons, not by reaching inside.

class BankAccount {
    private double balance;

    public double Balance {
        get { return balance; }
        set { if (value >= 0) balance = value; }
    }
}

Polymorphism

Polymorphism means "many forms." It allows one method name to behave differently depending on the context. There are two flavors:

  • Compile-time (method overloading) — same name, different parameters
  • Run-time (method overriding) — child class redefines parent's method using virtual/override

16. Methods

Methods are reusable blocks of code that do a specific job. Instead of copying the same logic everywhere, you write it once as a method and call it whenever needed. This saves time, reduces bugs, and makes your code readable.

static int Add(int a, int b) {
    return a + b;
}

int result = Add(12, 23);  // result = 35

Method Overloading

Same method name, but different parameter lists. The compiler picks the right version based on what you pass in. This is compile-time polymorphism.

int Add(int a, int b)         { return a + b; }
int Add(int a, int b, int c)  { return a + b + c; }
double Add(double a, double b) { return a + b; }

Method Overriding

A derived class replaces the parent's method with its own version. The parent method must be virtual, and the child uses override. This enables run-time polymorphism.

Method Hiding

Uses the new keyword to hide a parent method instead of overriding it. The hidden method is chosen based on the reference type, not the object type.

17. Collections

Collections are containers for groups of objects with built-in methods for adding, removing, searching, and sorting. They live in System.Collections.

ArrayList

A dynamic array that grows as you add items. Can hold any type of object (not type-safe).

Hashtable

Stores key-value pairs. You look up values by their key (like a dictionary). Keys must be unique and non-null.

Stack (LIFO)

Like a stack of plates — the last item added is the first one removed. Methods: Push(), Pop(), Peek().

Queue (FIFO)

Like a line at a store — the first item added is the first one removed. Methods: Enqueue(), Dequeue(), Peek().

19. Indexers

Indexers let you use an object like an array using square bracket syntax obj[index]. They're like properties, but with parameters (the index).

class Shelf {
    private string[] books = new string[3];

    public string this[int i] {
        get { return books[i]; }
        set { books[i] = value; }
    }
}

Shelf s = new Shelf();
s[0] = "C# in Depth";  // uses indexer

20. Interfaces

An interface is a contract. It says "any class that implements me must have these methods" but doesn't say how. It defines what to do, not how to do it. This is key to achieving multiple inheritance in C# (which doesn't support it with classes).

📚
Rules: All members are public and abstract by default. No fields allowed. A class can implement multiple interfaces. Convention: prefix names with I (e.g., IDisposable).
interface IAnimal {
    void Speak();
}

class Cat : IAnimal {
    public void Speak() { Console.WriteLine("Meow!"); }
}

Cat c = new Cat();
c.Speak();  // Meow!

21. Multithreading

Imagine you're cooking dinner while doing laundry. That's multithreading — doing multiple things simultaneously. Each thread is an independent path of execution. This is essential for responsive apps (so the UI doesn't freeze while loading data).

using System.Threading;

Thread t1 = new Thread(() => {
    for(int i = 0; i < 5; i++)
        Console.WriteLine("Thread 1: " + i);
});

Thread t2 = new Thread(() => {
    for(int j = 0; j < 5; j++)
        Console.WriteLine("Thread 2: " + j);
});

t1.Start();
t2.Start();
⚠️
Thread execution order is unpredictable — the output may vary each time you run the program. Use Thread.Sleep() to pause a thread, and synchronization tools (like lock) to prevent conflicts.

22. Regular Expressions

Regex is a pattern-matching language for text. Need to check if an email is valid? Find all phone numbers in a document? Extract dates from a log file? Regex is your tool. In C#, use the Regex class from System.Text.RegularExpressions.

using System.Text.RegularExpressions;

string pattern = @"^[0-9]{10}$";   // exactly 10 digits
Regex re = new Regex(pattern);

Console.WriteLine(re.IsMatch("9925612824"));   // True
Console.WriteLine(re.IsMatch("123"));          // False

Key methods: IsMatch() checks for a match, Match() returns the first match, Replace() substitutes matches, Split() breaks text at matches.

23. Exception Handling

No matter how careful you are, things go wrong — a file doesn't exist, a network call fails, someone divides by zero. Exception handling lets you gracefully deal with errors instead of letting your program crash.

KeywordWhat It Does
tryWraps code that might fail
catchHandles the error if one occurs
finallyRuns no matter what (error or not) — great for cleanup
throwManually triggers an exception
int[] arr = { 1, 2, 3 };

try {
    Console.WriteLine(arr[10]);  // index doesn't exist!
}
catch (IndexOutOfRangeException e) {
    Console.WriteLine("Oops: " + e.Message);
}
finally {
    Console.WriteLine("This always runs.");
}
Oops: Index was outside the bounds of the array.
This always runs.
✅
Pro tip: Catch specific exceptions (like DivideByZeroException) rather than a generic Exception. This way you handle each error type appropriately.
