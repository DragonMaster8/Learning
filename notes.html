01. What is C#?

Think of C# (pronounced "C-sharp") as Microsoft's answer to Java. It's a modern, object-oriented programming language that lets you build just about anything — from websites and desktop apps to mobile apps and video games. Microsoft created it, and some of the biggest names in tech (Microsoft, Stack Overflow, Unity) rely on it every single day.

The latest version is C# 12.0, and it keeps getting better with each release. If you already know a bit of C, C++, or Java, you'll feel right at home.

02. Writing Your First Program

Every programmer's journey starts with "Hello, World!" — it's a tradition. Here's what that looks like in C#. Don't worry if the syntax looks unfamiliar; we'll break everything down in later sections.

using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}
Hello, World!
💡
What's happening here? The using System; line imports built-in tools. namespace organizes your code. class is a container for methods. Main() is the entry point — the very first thing that runs.

03. A Quick History

C# was born out of necessity. In the late 1990s, Microsoft needed a language combining the power of C++ with the simplicity of Visual Basic. Anders Hejlsberg (who also created Turbo Pascal and TypeScript) led the team that designed it.

  • Developed under Microsoft's .NET initiative
  • Standardized by ECMA and ISO
  • First version released in 2002 alongside Visual Studio .NET
  • Closely related to C/C++ and Java in syntax and philosophy
⚠️
Keep in mind: C# 12.0 (released November 2023) is the latest version. Each new version adds features that make coding easier and more expressive.

04. .NET Framework & Its Components

Before your C# code can run, it needs a "platform" to live on. That platform is .NET — a software framework by Microsoft that acts like a middleman between your code and the operating system. Think of it as a translator that takes your C# instructions and turns them into something the computer actually understands.

.NET supports 60+ programming languages, but C# and VB.NET are the main ones. The first version (1.0) shipped in 2002.

Common Language Runtime (CLR)

The CLR is the engine of .NET. When you run a C# program, the CLR actually executes it. It doesn't care which .NET language you wrote in — it treats them all the same. It also handles memory, security, and errors behind the scenes.

  • Managed Code — Runs inside the CLR with garbage collection and type safety.
  • Unmanaged Code — Runs directly on the OS, bypassing CLR. Faster but riskier.

Framework Class Library (FCL)

Imagine a massive toolbox filled with pre-built tools for every common task — reading files, connecting to databases, making web requests. That's the FCL. It's a huge collection of reusable classes and methods you can plug into your code instead of building everything from scratch.

Garbage Collector (GC)

In C++, you manually free memory (and forgetting causes memory leaks). C# solves this with a Garbage Collector that automatically detects objects you're no longer using and reclaims that memory. It runs on the Managed Heap with an internal optimization engine.

Just-In-Time (JIT) Compiler

Here's how your code becomes executable: First, the C# compiler converts source code into CIL (Common Intermediate Language) — platform-independent instructions. Then at runtime, the JIT compiler converts CIL into native machine code optimized for your specific computer. Like a translator who speaks your computer's exact dialect.

05. Visual Studio

Visual Studio is Microsoft's all-in-one IDE (Integrated Development Environment). It's where you write, debug, and test your C# code — a super-powered text editor that understands your code, highlights errors, and suggests fixes.

  • Available for Windows and macOS

07. C# Fundamentals

Identifiers

An identifier is simply a name you give to things — classes, methods, variables. Think of them as labels on boxes telling you what's inside.

  • Can contain letters (A-Z, a-z), digits (0-9), and underscore (_)
  • Must NOT start with a digit
  • Cannot be a reserved keyword (unless prefixed with @)
  • Case-sensitive; max 512 characters
public class Student {        // "Student" is an identifier
    static void Main()         // "Main" is an identifier
    {
        int age = 21;            // "age" is an identifier
    }
}

Keywords

Keywords are reserved words that C# has claimed for itself. They have special meanings and you can't use them as your own names. C# has 78 keywords in total.

Common ones: using, namespace, class, public, private, static, void, int, string, bool, if, else, for, while, return, new, this, null.

Variables

A variable is like a labeled container that holds data. You put something in, give it a name, and refer to it later.

char grade = 'A';        // single character
int score = 95;          // whole number
double price = 19.99;    // decimal number
string name = "Alice";   // text
bool isPassed = true;    // true or false

Two ways to initialize: Compile-time (value set in code) or Run-time (value comes from user input during execution).

Literals

Literals are fixed values written directly in your code. When you write int x = 100;, the 100 is a literal.

Enumerations (enum)

Enums create a named set of constants. Instead of remembering "0 = Sunday," you write Day.Sunday and your code becomes self-documenting.

enum Month { Jan, Feb, Mar, Apr, May }
// Jan=0, Feb=1, Mar=2, Apr=3, May=4 (auto-assigned)

Console.WriteLine((int)Month.Mar);  // Output: 2
📚
By default, first member = 0, each successive one increments by 1. You can override this: enum Month { Jan = 1, Feb, Mar } makes Jan=1, Feb=2, Mar=3.

08. Decision Making Statements

Just like in real life, your program needs to make choices. "If the user is logged in, show the dashboard. Otherwise, show the login page." The main types: if, if-else, if-else-if ladder, and nested if.

int i = 20;

if (i == 10)
    Console.WriteLine("i is 10");
else if (i == 20)
    Console.WriteLine("i is 20");     // This runs!
else
    Console.WriteLine("not found");
i is 20

09. Switch Statement

When you have many values to check, an if-else-if chain gets ugly. The switch is cleaner — it takes one expression and jumps directly to the matching case.

📚
Rules: No duplicate cases. Values must be constants. Every case needs a break. The default is optional but recommended.
int day = 3;
switch (day) {
    case 1: Console.WriteLine("Monday");    break;
    case 2: Console.WriteLine("Tuesday");   break;
    case 3: Console.WriteLine("Wednesday"); break;
    default: Console.WriteLine("Other");   break;
}
Wednesday

10. Loops

Loops tell the computer to repeat something. Instead of writing the same line 100 times, write it once inside a loop.

  • Entry-controlled (while, for) — Condition checked first. May never run.
  • Exit-controlled (do-while) — Runs at least once, then checks.

while Loop

int x = 1;
while (x <= 4) {
    Console.WriteLine("Count: " + x);
    x++;
}

do-while Loop

int x = 100;
do {
    Console.WriteLine("Runs once!");
    x++;
} while (x < 5);  // false, but body already ran

for Loop

for (int i = 0; i < 5; i++)
    Console.WriteLine("Iteration: " + i);

foreach Loop

The friendliest loop — walks through every item without needing an index.

int[] nums = { 10, 20, 30, 40 };
foreach (int n in nums)
    Console.WriteLine(n);

11. Jump Statements

Sometimes you need to break out of the normal flow — exit a loop early, skip an iteration, or jump elsewhere.

break

Immediately exits the loop. Like hitting the emergency stop.

for (int i = 1; i <= 10; i++) {
    if (i == 5) break;
    Console.Write(i + " ");   // 1 2 3 4
}

continue

Skips the rest of the current iteration and jumps to the next one.

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;
    Console.Write(i + " ");   // 1 2 4 5
}

goto

Jumps to a labeled statement. Useful in switch blocks but discouraged elsewhere.

return

Exits the current method and optionally sends a value back to the caller.

12. Arrays

An array is a collection of items of the same type, stored in a row. Think of it as a row of mailboxes — each has a number (index) starting from 0.

string[] days = { "Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat" };
Console.WriteLine(days[0]);      // Sun
Console.WriteLine(days.Length);  // 7
Sun Mon Tue Wed Thu Fri Sat
⚠️
Array sizes are fixed once created. Need a resizable collection? Use List<T> instead.

13. Strings

A string is simply text. In C#, strings are objects of System.String and they are immutable — any "modification" creates a new string.

string s1 = "Hello";
String s2 = "World";
string full = s1 + " " + s2;  // "Hello World"
💡
string (lowercase) and String (uppercase) are identical — both alias System.String.

15. Object-Oriented Programming

OOP organizes code around objects rather than actions. The four pillars: Encapsulation, Inheritance, Polymorphism, and Abstraction.

Class & Object

A class is a blueprint (architect's plan). An object is a specific instance (an actual house). Define the structure once, then create as many objects as you need.

public class Dog {
    public string Name;
    public int Age;

    public Dog(string name, int age) {
        Name = name; Age = age;
    }
}
Dog buddy = new Dog("Buddy", 3);

Constructors

A special method that runs automatically when you create an object. Same name as the class, no return type.

Destructors

Opposite of constructors — run when an object is garbage collected. Defined with ~ClassName(). Rarely needed because GC handles cleanup.

Inheritance

Create a new class based on an existing one. The child gets all parent features for free and can add its own.

class Animal {
    public void Eat() { Console.WriteLine("Eating..."); }
}
class Dog : Animal {
    public void Bark() { Console.WriteLine("Woof!"); }
}
Dog d = new Dog();
d.Eat();   // inherited
d.Bark();  // own method

Encapsulation

Hiding internal details and exposing only what's necessary. Make variables private, provide controlled access through properties. Like a vending machine — you interact through buttons, not by reaching inside.

Polymorphism

Means "many forms." One method name, different behaviors depending on context:

  • Compile-time (method overloading) — same name, different parameters
  • Run-time (method overriding) — child redefines parent's method via virtual/override

16. Methods

Methods are reusable blocks of code. Write the logic once, call it whenever needed.

static int Add(int a, int b) {
    return a + b;
}
int result = Add(12, 23);  // 35

Method Overloading

Same name, different parameter lists. Compiler picks the right version. Compile-time polymorphism.

int Add(int a, int b)         { return a + b; }
int Add(int a, int b, int c)  { return a + b + c; }

Method Overriding

Derived class replaces parent's method. Parent uses virtual, child uses override. Run-time polymorphism.

Method Hiding

Uses new keyword to hide a parent method instead of overriding it. Also called Method Shadowing.

17. Collections

Collections are containers for groups of objects with built-in methods for adding, removing, and searching. Live in System.Collections.

ArrayList

A dynamic array that grows as you add items. Can hold any type (not type-safe).

Hashtable

Stores key-value pairs. Look up values by key. Keys must be unique and non-null.

Stack (LIFO)

Like a stack of plates — last in, first out. Methods: Push(), Pop(), Peek().

Queue (FIFO)

Like a line at a store — first in, first out. Methods: Enqueue(), Dequeue(), Peek().

19. Indexers

Indexers let you use an object like an array with square bracket syntax obj[index].

class Shelf {
    private string[] books = new string[3];
    public string this[int i] {
        get { return books[i]; }
        set { books[i] = value; }
    }
}
Shelf s = new Shelf();
s[0] = "C# in Depth";

20. Interfaces

An interface is a contract: "any class that implements me must have these methods." Defines what to do, not how. Enables multiple inheritance.

📚
Rules: All members public and abstract by default. No fields. A class can implement multiple interfaces. Convention: prefix with I (e.g., IDisposable).
interface IAnimal { void Speak(); }

class Cat : IAnimal {
    public void Speak() { Console.WriteLine("Meow!"); }
}

21. Multithreading

Imagine cooking dinner while doing laundry. That's multithreading — multiple things simultaneously. Essential for responsive apps so the UI doesn't freeze.

using System.Threading;

Thread t1 = new Thread(() => {
    for(int i = 0; i < 5; i++)
        Console.WriteLine("Thread 1: " + i);
});
t1.Start();
⚠️
Thread execution order is unpredictable. Use lock for synchronization and Thread.Sleep() to pause.

22. Regular Expressions

Regex is a pattern-matching language for text. Validate emails, find phone numbers, extract data. Use Regex from System.Text.RegularExpressions.

string pattern = @"^[0-9]{10}$";   // exactly 10 digits
Regex re = new Regex(pattern);
re.IsMatch("9925612824");   // True
re.IsMatch("123");          // False

Key methods: IsMatch(), Match(), Replace(), Split()

23. Exception Handling

Things go wrong — files don't exist, networks fail, someone divides by zero. Exception handling lets you gracefully deal with errors instead of crashing.

int[] arr = { 1, 2, 3 };
try {
    Console.WriteLine(arr[10]);
}
catch (IndexOutOfRangeException e) {
    Console.WriteLine("Oops: " + e.Message);
}
finally {
    Console.WriteLine("This always runs.");
}
Oops: Index was outside the bounds of the array.

✅
Pro tip: Catch specific exceptions (like DivideByZeroException) rather than a generic Exception.
